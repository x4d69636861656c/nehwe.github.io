# nehwe-tech Hugo Theme

A cyberpunk-styled, one-page Hugo theme built for **nehwe.tech** — security
consulting, vulnerability scanning, compliance, and AI automation. Cyan means
"scanning / clear," red means "vulnerability found" — the palette tracks the
actual subject matter, not just decoration.

## 1. Install it

Copy this entire folder into your existing site's theme directory so you have:

```
your-site/
  themes/
    nehwe-tech/   <- this folder
```

Then in your site's `hugo.toml` (or `config.toml`), set:

```toml
theme = "nehwe-tech"
```

## 2. Bring in the example config + content

This repo's `exampleSite/` is a full working starting point.

- Merge the `[params]` block and the `[[menu.main]]` entries from
  `exampleSite/hugo.toml` into your own site's config.
- Copy `exampleSite/content/_index.md` to your own `content/_index.md` and
  edit the copy to match your real services, certs, and stats.

Then run, from your site root:

```bash
hugo server -D
```

and open `http://localhost:1313`.

## 3. Edit content — no templates required

Everything visible on the homepage lives in `content/_index.md` front matter:

| Section      | Front matter key | Controls |
|---|---|---|
| Hero         | `hero`        | headline, subhead, CTA buttons, terminal animation script |
| Services     | `services`    | the 3 (or more) service cards |
| Compliance   | `compliance`  | the framework badges (NIST, CMMC, ISO, PCI DSS, HIPAA...) |
| Process      | `process`     | the numbered "how a scan runs" steps |
| About        | `about`       | bio paragraphs + the stats sidebar |
| Contact      | `contact`     | closing heading, subtext, and email |

Add or remove items from any `items` / `steps` / `paragraphs` / `stats` list
freely — the templates loop over whatever is there.

Site-wide things (nav links, brand name, contact email, LinkedIn, the nav
button) live in `hugo.toml` under `[params]` and `[[menu.main]]`, not in
content.

## 4. Customize the look

All colors and fonts are CSS variables at the top of `static/css/style.css`:

```css
--void        /* page background */
--panel       /* card/panel background */
--cyan        /* primary accent — "scanning / clear" */
--alert       /* secondary accent — "vulnerability found" */
--text        /* primary text */
--text-dim    /* muted text */
--mono        /* display/terminal font */
--sans        /* body font */
```

Change a value here and the whole site re-themes — no need to touch any
template.

## 5. Add a blog (optional)

The theme ships generic `list.html` / `single.html` templates styled to match.
Just create `content/blog/`, drop in `.md` posts, and add a "Blog" entry to
`[[menu.main]]` pointing at `/blog/`.

## 6. Deploy to GitHub Pages

Since you're already on GitHub Pages, the cleanest path is a GitHub Actions
workflow that builds with Hugo on every push. Hugo's own docs have a
ready-made workflow file for exactly this:
https://gohugo.io/host-and-deploy/host-on-github-pages/

Drop that workflow into `.github/workflows/hugo.yml` in your site repo, set
your repo's Pages source to "GitHub Actions" (Settings → Pages), and pushes to
your default branch will build and deploy automatically — no `public/` folder
to manage by hand.
